# Foggy LLM 语义层白皮书 v1.0 发布说明

> 文档版本：v1.0
>
> 发布状态：公开冻结版本
>
> 版本定位：这是 Foggy 面向 LLM 语义层引擎方向的第一版公开白皮书与参考文档集合。它描述当前已经实现并经过验证的能力边界，不等同于 Java 引擎、Python 引擎或 Odoo 集成包的包版本号。

## 1. 本版本包含什么

v1.0 围绕当前已经实现和验证过的核心能力组织：

- TM/QM 语义建模：用表模型和查询模型把物理数据结构收敛为业务语义契约。
- JSON Query DSL：通过结构化查询对象表达筛选、排序、分页、聚合、时间窗口、层级查询、pivot 和小计等能力。
- Compose 组合式分析：通过多步骤查询计划支持更复杂的分析流程。
- MCP 工具接入：通过 `dataset.list_model`、`dataset.describe_model_internal`、`dataset.query_model` 和 `dataset.compose_script` 暴露可治理的查询入口。
- 治理与证据：由引擎执行字段校验、权限切片、查询编译、方言适配和查询证据输出。
- 产品集成案例：以 Foggy Odoo Bridge Pro 和社区版集成为当前 ERP 场景参考。

## 2. 本版本不包含什么

v1.0 不把以下内容作为当前稳定公开能力：

- Virtual Semantic SQL。
- 面向临时结果集的 SQL 二次检索。
- 向量检索类 DSL 操作符。
- 自动保证业务分析结论正确的能力。
- 通用 BI 仪表板、报表设计器或完整可视化产品能力。

这些方向可以进入后续版本或单独能力文档，但不会继续追加到已经冻结的 v1.0 白皮书中。

## 4. 版本冻结规则

v1.0 发布后只做以下类型的维护：

- 修正错误描述。
- 修正失效链接。
- 修正术语不一致。
- 补充风险边界或兼容性说明。
- 对已经实现能力补充更准确的用法说明。

以下变更不进入 v1.0：

- 新增未发布能力。
- 把路线图能力改写成当前稳定功能。
- 为了营销表达扩大能力边界。
- 引入未经验证的 benchmark 数字或保证性结论。

## 5. 版本号说明

白皮书版本是公开语义契约版本。它用于稳定对外口径和能力边界，不要求与实现包版本号一一相同。

当前 Java 引擎、Python 引擎和产品集成包可能使用不同的历史版本号。后续可以通过兼容性矩阵说明各实现版本与公开语义契约版本之间的对应关系。

## 6. 后续方向

后续版本重点评估以下方向：

- Virtual Semantic SQL 作为 DSL 上层契约：由受限 SQL 解析为 AST，再转换为 Foggy 可治理的 DSL 或执行计划。
- 二次结果集分析：把中间结果放入受控的内存数据库，允许继续使用 DSL 或受限 SQL 做二次检索。
- DSL、Compose、CTE 和 SQL 的边界评估：根据任务类型、模型能力、错误率、可解释性、权限边界和执行证据持续调整。
- Skill / Playbook：沉淀可复用的高阶分析流程，提升复杂分析的一致性。
- Benchmark 证据补充：补充真实场景的任务集、失败模式、空结果处理、模型差异和人工复核结论。

这些方向属于后续演进，不改变 v1.0 当前能力边界。
